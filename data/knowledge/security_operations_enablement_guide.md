# Security Operations Certification Enablement Guide (SYNTHETIC)

> Source ID: KB-SEC-001 — approved internal security certification reference (fabricated for demo).

## Role-to-Certification Map

- **SOC Analyst** — Primary: SC-200, Secondary: SC-100
- **Detection Engineer** — Primary: SC-200, Secondary: AZ-500
- **Incident Responder** — Primary: SC-200
- **Security Engineer** — Primary: AZ-500, Secondary: SC-200

## SC-200 — Security Operations Analyst Associate

### Domain: Defender XDR (35% of exam)
Learners must investigate incidents across Microsoft Defender XDR, correlate alerts, and understand how incidents move from detection to triage and remediation.

### Domain: Defender for Cloud (25% of exam)
Learners must interpret cloud security posture, investigate recommendations, and connect workload-protection signals to incident response decisions.

### Domain: Microsoft Sentinel (40% of exam)
Microsoft Sentinel is the heaviest SC-200 domain in this synthetic blueprint. Learners must understand analytics rules, KQL investigation, incident queues, and workbook-driven detection review.

## Readiness Bar

Target at least **75% practice score across every SC-200 exam domain** before sitting the exam. A passing learner can explain a Sentinel incident, write a focused KQL query, and choose the next response action from Defender XDR and Defender for Cloud evidence.

## Common Misconceptions

- Treating Microsoft Sentinel as only a log store instead of a detection and incident-management platform.
- Studying only the domain with the highest current score instead of the domain with the highest exam weight.
- Using total average score as readiness proof while one weighted domain remains below the 75% gate.
